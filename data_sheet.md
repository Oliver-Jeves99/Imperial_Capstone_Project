Datasheet
Motivation

For what purpose was the dataset created?
The dataset was created as part of the Black-Box Optimisation (BBO) capstone challenge. Its purpose was to simulate real-world optimisation problems where the underlying function is unknown and expensive to evaluate. The dataset represents sequential query results generated during the optimisation process, allowing participants to iteratively improve their search strategies.

Who created the dataset and who funded it?
The dataset was generated through the capstone project platform used in the programme. The underlying functions were provided by the course organisers to simulate unknown optimisation problems. Participants generated the data themselves by submitting query points and receiving corresponding output values. The educational programme hosting the capstone project funded and managed the system.

Composition

What do the instances represent?
Each instance represents a query submitted to one of eight unknown functions. Each query contains:

A set of input variables (between 2 and 8 dimensions depending on the function)

A corresponding output value returned by the system

The dataset therefore consists of pairs of:

Input vector → function output

Each function represents a separate optimisation problem.

How many instances are there?
Each participant was allowed one query per function per week. Over the course of the capstone project this resulted in multiple observations per function, typically around 12–13 queries per function.

In total the dataset consists of:

8 optimisation problems

~12–13 observations per function

Approximately 96–104 total query observations.

Is there any missing data?
No missing values are present. Each submitted query returned a valid numerical output.

Does the dataset contain confidential data?
No. The dataset contains only synthetic numerical inputs and outputs generated by the capstone platform. It does not contain personal, private, or confidential information.

Collection Process

How was the data acquired?
The data was collected through the capstone project portal. Participants submitted input queries for each function, and the platform evaluated these queries against hidden functions and returned the output values.

Sampling strategy
The dataset was collected sequentially through iterative optimisation. Early queries were exploratory to understand the function landscape, while later queries focused on exploiting promising regions discovered during the optimisation process.

Time frame of data collection
The dataset was collected weekly over the duration of the capstone project.

Preprocessing / Cleaning / Labelling

Minimal preprocessing was required.

Steps included:

Organising weekly query results into structured datasets

Grouping observations by function

Formatting input vectors consistently

Ensuring numeric precision (values reported to six decimal places)

No labels were added because the task involved optimisation rather than classification.

The original query results from each week were retained to preserve the raw optimisation history.

Uses

What tasks could the dataset be used for?

Studying black-box optimisation strategies

Testing Bayesian optimisation algorithms

Comparing exploration vs exploitation strategies

Training surrogate models such as Gaussian Processes

Educational demonstrations of sequential optimisation

Potential risks or considerations

Because the dataset is extremely small and sequentially generated, models trained on it may easily overfit. Additionally, the data reflects optimisation decisions rather than unbiased sampling of the search space.

Users should therefore treat the dataset as an optimisation trace rather than a representative dataset.

Tasks for which the dataset should not be used

Benchmarking general machine learning models

Training large predictive models

Drawing conclusions about real-world systems

The dataset is best suited for studying optimisation behaviour rather than general predictive modelling.

Distribution

How has the dataset already been distributed?
The dataset was distributed through the capstone project portal during the course. Participants accessed their own query results after each weekly submission.

Licensing and terms of use
The dataset is part of the course material and subject to the educational platform’s usage policies.

Maintenance

The dataset is maintained by the course organisers and the capstone project platform. Individual participants maintain their own copies of their query histories.
